## 4. Boot / Reset Related Nets

From the main module area:

| Function | Net / GPIO | Circuit notes |
|---|---|---|
| Boot mode | `BOOTMODE` / GPIO35 | Pulled up to `VCC3V3` via R34 `10K`; connected to SW1 |
| Chip enable | `CHIP_PU` | Pulled up via R37 `10K`, capacitor C38 `105/10V`, connected to SW2 |
| C6 chip enable | `C6_CHIP_PU` | Pulled up via R81 `10K`, capacitor C64 `105/10V` |
| C6 IO9 | `C6_IO9` | Pulled up to `VCC3V3` via R80 `5K1` |

Relevant ESP32-P4 boot strapping from TRM:

- GPIO35 high at reset selects default SPI Boot mode.
- This schematic pulls GPIO35 / `BOOTMODE` up, so default boot should be SPI Boot unless SW1 or external circuitry pulls it low.

## 16. Rust / Bare-metal Development Notes

Useful board-level implications:

1. **Default boot mode**  
   GPIO35 / `BOOTMODE` is pulled up, matching ESP32-P4 default SPI Boot mode.

2. **UART0 pins**  
   UART0 appears on:
   - GPIO37: `UART0_TXD`
   - GPIO38: `UART0_RXD`

3. **USB options**  
   There are two Type-C sections:
   - Full-Speed USB on `USB1P1_P/N`
   - High-Speed / ESP USB on `ESP_USB_P/N`

4. **microSD**  
   SD card uses `SD_DATA0..3`, `SD_CMD`, `SD_CLK`, with TF power controlled/switched through `TF_VCC` circuitry.

5. **Display and camera**  
   DSI and CSI lanes are routed directly from the ESP32-P4 module to FPC connectors.

6. **Audio**  
   ES8311 codec is controlled by I2C and I2S; speaker output is amplified by NS4150 from `VOUT-BAT`.

7. **Expansion header**  
   JP1 exposes many Ethernet/RMII-related GPIOs plus C6 UART/control lines and I2C.
